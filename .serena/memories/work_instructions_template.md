## 🚨 特別注意事項

### 安全性考量
- 不要包含敏感資訊
- 確認檔案權限設定正確
- 避免覆蓋重要的現有檔案

### 相容性考量
- 確保與現有程式碼相容
- 考慮不同作業系統的相容性
- 注意版本相依性問題

### 效能考量
- 避免不必要的資源浪費
- 考慮程式碼執行效率
- 適當的記憶體使用

## 🔄 狀態更新要求

### 執行過程中的更新
- 每完成一個主要步驟都要更新狀態
- 遇到問題時立即記錄
- 產出檔案後立即記錄到工作紀錄

### 狀態更新格式
```
[時間] Cursor [狀態] [簡短描述]
```

### 完成後的更新
- 標記任務為完成狀態
- 列出所有產出的檔案
- 記錄遇到的問題和解決方案
- 執行基本的自我檢查

## 🆘 問題處理

### 遇到技術問題時
1. 先嘗試查閱相關文檔
2. 檢查是否為常見問題
3. 記錄問題的詳細描述
4. 向 Claude Code 回報並等待指引

### 需求不明確時
1. 記錄具體的疑問點
2. 暫停執行並向 Claude Code 詢問
3. 等待明確指引後再繼續

## 📞 溝通協定

### 定期回報
- 每小時更新一次進度（如任務較長）
- 遇到阻礙時立即回報
- 完成重要里程碑時主動回報

### 回報格式
簡短且明確，包含：
- 當前狀態
- 已完成的工作
- 下一步計畫
- 需要的協助（如有）

## 使用說明
1. Claude Code 根據任務需求填寫此範本
2. 儲存到 Serena Memory 供 Cursor 讀取
3. Cursor 按照指引執行並更新狀態
4. Claude Code 監控進度並提供必要支援
